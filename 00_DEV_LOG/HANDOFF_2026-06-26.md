## THE OPEN PROBLEM — WORKFLOW TRIGGERS NOT FIRING

### What happened
Workflows were imported via n8n UI with encoding errors (mojibake em dashes in JSON names).
JSON names were cleaned. Workflows re-imported. DB shows 3 workflows active=true with correct node code.
But test task dropped into QUEUE/ was not picked up.

### Root cause (diagnosed)
n8n 2.x ships with localFileTrigger DISABLED by default.
Re-enabling requires two env vars set BEFORE n8n starts:
  $env:NODES_EXCLUDE = "[]"
  $env:N8N_ENABLE_LOCAL_FILE_NODE = "true"

These vars exist in start_n8n.ps1 BUT NOT in n8n_env.ps1.
This session restarted n8n by dot-sourcing n8n_env.ps1 only — so the running instance
has localFileTrigger disabled. File watcher never registers. Tasks don't fire.

### The fix (Block F — do this first)
Merge start_n8n.ps1 env vars into n8n_env.ps1 so any restart has them.
Then restart n8n by dot-sourcing n8n_env.ps1 as the single source of truth.
Then toggle each workflow off and back on in the UI (or via API) to force trigger re-registration.
