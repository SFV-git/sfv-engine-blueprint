# SFV ENGINE — BLUEPRINT VAULT

This vault is the blueprint brain for SFV Engine.
It is NOT the Engine itself. It documents what the Engine is,
how it works, and how to build it.

## RULES
- Nothing gets built until it is tagged STATUS: CANON
- Everything unclear is tagged UNCONFIRMED
- Everything proposed by Claude is tagged FOR HUMAN REVIEW
- Will is the final authority on all decisions

## HOW TO USE
- Open DASHBOARD.md for system status at a glance
- Open 00_DEV_LOG/QUESTIONS_FOR_WILL.md for open decisions
- Open FOR_HUMAN_REVIEW/PROPOSALS.md for Claude proposals
- Open 11_VERSIONS/ROADMAP.md for where we are going

## VAULT STRUCTURE
- 00_DEV_LOG/ — decisions, questions, unconfirmed items
- 01_CANON_RULES/ — locked rules governing the build
- 02_BRANCHES/ — every SFV branch definition
- 03_INFRASTRUCTURE/ — storage, naming, paths, environment
- 04_WORKFLOWS/ — ingest, cull, export, delivery, archive
- 05_AI_LAYER/ — model routing, local models, quality control
- 06_TOOLS/ — tool stack, status, integrations
- 07_SCALING/ — operator model, branch independence, nationwide
- 08_TESTS/ — paper trials, failure tests, build readiness
- 09_PROMPTS/ — prompts for every AI tool in the system
- 10_REFERENCES/ — research, case studies, external links
- 11_VERSIONS/ — roadmap, version log, upgrade checkpoints
- 12_DATABANKS/ — content, taste, client, brand, training data
- 99_INBOX/ — raw ideas, extracts, scratchpad
- FOR_HUMAN_REVIEW/ — Claude proposals awaiting Will approval
